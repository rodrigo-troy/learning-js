• Functions
    • Functions are first-class objects that are treated just like any other objects within JavaScript. Similar to any other object type, they can be:
      • Created via literals
      • Assigned to variables or properties
      • Assigned to arrays entries
      • Passed as parameters
      • Returned as function results
      • Assigned properties and methods
    • Every function receives 2 additional parameters:
      • this
      • argument
    • The "this" parameter represents the function context, the object on which our function is invoked, whereas the arguments parameter represents all arguments that are passed in through a function call.
    • Callback functions are functions that other code will later "call back", and are often used, especially with event handling.
    • There is no runtime error when the number of arguments and the number of parameters don't match.
    • If more arguments are supplied than there are parameters, the “excess” arguments aren’t assigned to parameter names.
    • The way in which a function is defined significantly influences when the function is available to be invoked and how it behaves, as well as on which object the function can be invoked.
    • The way in which we invoke a function has a great influence on how the implicit function parameters are determined.
    • Memoization allows the function to remember previously computed values, thereby improving the performance of subsequent invocations.
